Preparations #

Please follow the steps for the best user experience:

  1. Download LaTEX for Windows on: https://www.tug.org/texlive/acquire-netinstall.html (https://www.tug.org/texlive/acquire-netinstall.html).

  2. Download the App on: https://github.com/saigkill/latex_curriculum_vitae-dotnet/releases (https://github.com/saigkill/latex_curriculum_vitae-dotnet/releases).

  3. Move the executable to the place, where you want (eg: Desktop or Userdirectory). The app will be executed there, without any installation.

  4. Launch the app.

User settings #

After launching latex_curriculum_vitae the first time, it opens a User settings window, where you can place your information.

Important

The Email settings are mandatory in case of you wish to send out the emails directly. The needed information about the SMTP host you get in your email-providers settings.

If you want to shorten your links before they saved, you can use BitLy. Just get a token via https://bitly.is/accesstoken (https://bitly.is/accesstoken) and place them in the BitLy Token Field.

The Save Button saves your changes and closes the window. The Close just closes the window without any changes in the config file. If you are using the button:

then you forwarded to the online documentation (this one).

During the firstrun of latex_curriculum_vitae installs a bunch of needed files at: %appdata%/latex_curriculum_vitae. In my case the full path looks like: C:\Users\Sasch\Appdata\Roaming\latex_curriculum_vitae. You also can type in Windows Explorers addressfield %appdata%/latex_curriculum_vitae.

The AppData directory hold all relevant data for building your job application. After the copy process is done, close the App, because you have sone things to do before producing a job application.

Your Appdata directory looks like:

Let's show what is important there. The first Directory contains 3 directories

  • Bibliography

  • Certificates

  • Certificates of Employment

The Bibliography directory only holds a file called bibliography.bib. This is a standard bibfile from Latex, currently filled with my publications. Feel free to replace their with yours.

Important

In case you are not using this feature so just comment out in the curriculum_vitae.tex file the part \addbibresource{bibliography.bib} and also the section: 

\section{Publikationen}
\printbibsection{article}{Artikel}
\printbibsection{book}{Bücher} 
\printbibsection{misc}{other publications}

The second directory Certificates is empty. In that directory, you can place all your certificates (maybe a further education or a course what you finished). The third directory is Certificates of Employment is reserved for that kind of certificates, what you become from your boss after ending your job. For the Certificates and the Certificates of Employment are some things important.

In the most countries you have to order your material anti-chronologic. That means the first certificate is the last one, ordered to the oldest certificate. To provide this, you have to add a Prefix for each certificate. Lets say, you have three certificates from jobs in 2018, 2019 and 2020. Then prefix the certificates so:

  • 2_your_2019job.pdf

Your mention is right: The next will be a 4_your_2021job.pdf. By preparing the files in that case, i can do a reverse ordering by the app itself.

Now we looking the other directories. We have a Curriculum_Vitae directory. And it contains a predefined LaTEX file for the curriculum vitae called curriculum_vitae.tex. Then we have a file friggeri-cv.cls. As you maybe expect, this is the class file for producing the curriculum vitae. The Letter of Application directory holds the letter_of_application.tex file, what we use for producing the letter of application. The Pictures directory contains some icons for building the cv and also a signature.png and a userpic.jpg. Just replace them with your own signature file and userpic and overwrite the delivered one.

In your AppData directory we also have two other files. Per default, the JobApplications.csv is empty by the first run, and will be filled during the creating part of the job application. You can open the csv file with a Excel or LibreOffice Calc.

Important

If you want to open and change the csv, make a copy, and leave the original one untouched.

The personal_data.tex holds your personal data. As you set your preferences in the User Settings Window, this file was produced.

Important

It will be newly created after each iteration. So if you want to change your data, use the User Settings Window.

Preparing the latex files #

As already written, we have the LaTEX files letter_of_application.tex and curriculum_vitae.tex. You have to modify it for your needs. Also it's possible to use your own texfiles. It is just important to save they as letter_of_application.tex and curriculum_vitae.tex in the prepared directories. This app produces two files, for compiling the job application: the first personal_data.tex you already heard. It looks in my case: 

\def\firstname{Sascha}
\def\familyname{Manns}
\def\mystreet{Maifeldstraße 10}
\def\mycity{56727 Mayen}
\def\myphone{+49-000-0000000}
\def\myemail{Sascha.Manns@outlook.de}
\def\myblog{http://saschamanns.de}

So if you just add a \input{personal_data.tex} to your Curriculum_vitae.tex you can use the newly defined macros. Each place in the document where you write \firstname, LaTEX sets your firstname. If you prepare a own cv, and want to add a image reference or link with a image, you have to set the full path like: \href{https://www.facebook.com/sascha.manns}{sascha.manns}\includegraphics[width=0.4cm]{$HOME/AppData/Roaming/latex_curriculum_vitae/Pictures/facebook.png}.

The second mentioned file is called application_details.tex. It will also generated by the app during the creating process. It looks like:

\def\jobtitle{Fachinformatiker Anwendungsentwicklung}
\def\company{XYZ GmbH}
\def\contact{Mr. Bean}
\def\street{Street 2}
\def\city{A City}
\def\salutation{Dear Mr. Bean,}
\def\subject{Application as Fachinformatiker Anwendungsentwicklung}
\def\addressstring{XYZ GmbH\\Mr. Bean \\Street 2\\56068 A City}

Because that file will jsut be created inside the temporary directory, where the compilation happens, you just can use \input{application_details.tex} to use the macros defined there.
